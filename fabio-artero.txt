1. Il comando 'git commit -m <commento>' permette 
di effettuare un commit esaustivo; la flag -m permette di inserire 
un commento. La staging area è una delle tre 
aree con cui git lavora. Nella staging area verranno
messi tutti quei file non ancora committati o non 
ancora trackati

2. Il comando 'git branch <nuovo branch>' permette di creare un nuovo branch. 
I branch sono una ramificazione del source tree originale che 
permette l'implementazioni indipendenti. L'HEAD, semplicemente, è 
un puntatore che permette la corretta individuazione del branch corrente.

3. Un repository remoto è come un database che archivia le modifiche 
apportate al codice nel tempo. Tiene traccia della cronologia del 
progetto e di chi ha apportato le modifiche, anche se sbagli qualcosa, 
puoi sempre tornare allo stato precedente. Il comando 'git branch -a' 
permette di visualizzare tutti i branch all'interno del repository 
(la flag -a indica all)
