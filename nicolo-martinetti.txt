1. Git add è un comando di git che permette di aggiungere
un file al tracking e quindi alla staging area
La staging area è una delle tre aree con cui git lavora.
Nella staging area verranno messi tutti quei file non
ancora committati o non ancora trackati.

2. Git branch è un comando di git che permette di creare
nuovi branch all'interno della repository. I branch sono
una ramificazione del source tree originale che permette
l'implementazioni indipendenti. L'HEAD, semplicemente,
è un puntatore che permette la corretta individuazione
del branch corrente.

3. Un repository remoto è come un database che 
archivia le modifiche apportate al codice nel tempo. 
Tiene traccia della cronologia del progetto e di chi 
ha apportato le modifiche, anche se sbagli qualcosa, 
puoi sempre tornare allo stato precedente. 
Git fetch è un comando di git che permette il controllo
e un eventuale aggiornamento della repository locale
con la repository remota. Questo permette l'allineamento
del progetto locale con quello remoto.
